👥 核心角色

消费级 eSIM 角色 (SGP.21/22)

🏢

MNO / 运营商

提供移动网络服务，通过ES2+向SM-DP+下单，提供激活码给用户

📦

SM-DP+

创建、加密、签名Profile包，绑定到特定eUICC，安全传输到设备

🔍

SM-DS

发现服务，接收SM-DP+的EID注册，向LPA返回SM-DP+地址

📱

LPA

本地配置助手：LDS(发现)、LPD(下载)、LUI(用户界面)

IoT eSIM 角色 (SGP.31/32)

☁️

eIM (新增!)

IoT远程管理器：服务器端触发Profile操作，支持批量设备管理

  • 触发Profile下载
  • 执行PSMO操作
  • eCO签名验证
  • 海量设备队列管理
🔧

IPA (替代LPA)

IoT配置助手：无UI或受限UI，支持低带宽连接

  • SM-DS发现服务
  • Profile代理下载
  • 通知转发
  • PSMO/eCO传递

eUICC 内部架构

  • ECASD — EUM Authentication Root SK, eUICC私钥, CI公钥
  • ISD-R — 发行方安全域根，创建和管理所有ISD-P
  • ISD-P — 每个Profile的安全容器，与SM-DP+安全通信
  • MNO-SD — 运营商OTA密钥和安全通道

🔄 核心流程

消费级 Profile 下载流程 (SGP.22)

1

Profile 准备

运营商通过ES2+向SM-DP+下单，SM-DP+创建Profile包，生成Bound Profile Package

2

SM-DS Event注册 (可选)

SM-DP+通过ES12注册Event Record (EID + SM-DP+ URL) 到SM-DS

3

用户触发

用户扫描运营商QR码，获取Activation Code，触发下载请求

4

SM-DS查询 / 获取SM-DP+地址

LPA通过ES11查询SM-DS，获取SM-DP+ URL

5

TLS双向认证

通过ES9+与SM-DP+建立安全连接，交换证书验证身份

6

Profile下载

Bound Profile Package通过ES9+传输到LPA，再通过ES8+传输到eUICC

7

eUICC验证安装

签名验证、绑定校验、解密Profile、安装到ISD-P

8

用户确认启用

用户通过LUI确认，Profile状态变为"已启用"，注册网络

IoT Profile 远程管理 (SGP.32)

1

eIM触发下载

服务器端通过ESipa向IPA发送下载命令，无需用户介入

2

IPA连接SM-DP+

通过ES9+建立连接，获取Bound Profile Package

3

Profile传输到eUICC

通过ES10b将Profile传输到eUICC

4

eUICC安装

验证签名、解密、安装到ISD-P

5

eIM执行PSMO

通过ES9+'或ESipa执行Enable/Disable/Delete操作

6

状态通知

通知通过SM-DP+回传给eIM，Profile正式启用

SGP.32 IoT 特有功能

⚡

Immediate Enable

Profile下载完成后立即启用，无需eIM额外操作，支持refreshFlag控制

🔄

Profile Rollback

回滚到之前启用的Profile，保证操作原子性

🔻

Fallback Mechanism

紧急情况激活备用Profile，支持ReturnFromFallback恢复

📦

Compact Notification

紧凑通知格式，减少低带宽场景的传输开销

⚖️ 消费级 vs IoT 对比

维度 消费级 (SGP.21/22) IoT (SGP.31/32)
触发方式 👤 用户发起 (Pull) ☁️ 服务器发起 (Push)
管理主体 👤 最终用户 🏢 企业/设备管理平台
设备交互 📱 丰富UI (QR码/APP) 📴 无UI或受限UI
规模 📱 单设备管理 🌐 海量设备批量管理
设备代理 LPA IPA
服务器角色 SM-DP+ SM-DP+ + eIM
典型场景 📱 手机/平板/手表 🔋 电表/车载/追踪器
连接方式 Wi-Fi/蜂窝初始连接 Bootstrap Profile
生命周期 用户自主控制 策略驱动自动化
传输优化 标准HTTPS 支持CoAP/UDP低带宽

规范演进路径

1

SGP.01/02 (M2M Legacy)

2013年，局限性：SMS强依赖、SM-SR耦合、复杂度高

2

SGP.21/22 (Consumer)

2016年，创新：LPA、SM-DS发现服务、用户Pull模式

3

SGP.31/32 (IoT Modern)

2022-2024年，融合消费级简洁 + eIM服务器管理 + Push模式

📝 总结

选型建议

场景 推荐规范 原因
📱 手机/平板/手表 SGP.21/22 用户交互、扫码激活
⌚ 智能手表(无完整UI) SGP.21/22 或 SGP.31/32 可用LPA或IPA
🚗 车联网/工业网关 SGP.31/32 服务器管理、大规模
🔋 智能电表/传感器 SGP.31/32 低功耗、无UI、NB-IoT
