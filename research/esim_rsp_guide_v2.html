👥 核心角色

🏢 MNO / 运营商

提供移动网络服务，通过ES2+向SM-DP+下单，提供激活码给用户

📦 SM-DP+

创建、加密、签名Profile包，绑定到特定eUICC，安全传输到设备

🔍 SM-DS

发现服务，接收SM-DP+的EID注册，向LPA返回SM-DP+地址

📱 LPA (消费级)

本地配置助手：LDS(发现)、LPD(下载)、LUI(用户界面)

☁️ eIM (IoT新增)

IoT远程管理器：服务器端触发Profile操作，支持批量设备管理

🔧 IPA (IoT)

IoT配置助手：替代LPA，无UI或受限UI，支持低带宽连接

🔄 核心流程

消费级 Profile 下载流程 (SGP.22)

1

Profile 准备

运营商通过ES2+向SM-DP+下单，SM-DP+创建Profile包

2

SM-DS Event注册

SM-DP+通过ES12注册Event Record到SM-DS

3

用户触发

用户扫描QR码，获取Activation Code

4

SM-DS查询

LPA通过ES11查询SM-DS，获取SM-DP+ URL

5

TLS双向认证

通过ES9+与SM-DP+建立安全连接

6

Profile下载

Bound Profile Package通过ES9+/ES8+传输到eUICC

7

eUICC验证安装

签名验证、解密、安装到ISD-P

8

用户确认启用

用户通过LUI确认，Profile状态变为"已启用"

Profile 生命周期状态机

📥 未下载 → 📁 已下载(禁用) → ✅ 已启用 ← ⛔ 禁用 ← 🗑️ 已删除

IoT Profile 远程管理 (SGP.32)

1

eIM触发下载

服务器端通过ESipa向IPA发送下载命令，无需用户介入

2

IPA连接SM-DP+

通过ES9+建立连接，获取Bound Profile Package

3

Profile传输

通过ES10b将Profile传输到eUICC

4

eUICC安装

验证签名、解密、安装到ISD-P

5

eIM执行PSMO

通过ES9+'或ESipa执行Enable/Disable/Delete

6

状态通知

通知通过SM-DP+回传给eIM

SGP.32 IoT 特有功能

⚡ Immediate Enable

Profile下载完成后立即启用，无需eIM额外操作

🔄 Profile Rollback

回滚到之前启用的Profile，保证操作原子性

🔻 Fallback Mechanism

紧急情况激活备用Profile

📦 Compact Notification

紧凑通知格式，减少低带宽传输开销

⚖️ 消费级 vs IoT 对比

维度消费级 (SGP.21/22)IoT (SGP.31/32)
触发方式👤 用户发起 (Pull)☁️ 服务器发起 (Push)
管理主体👤 最终用户🏢 企业/设备管理平台
设备交互📱 丰富UI (QR码/APP)📴 无UI或受限UI
规模📱 单设备管理🌐 海量设备批量管理
设备代理LPAIPA
服务器角色SM-DP+SM-DP+ + eIM
典型场景📱 手机/平板/手表🔋 电表/车载/追踪器
传输优化标准HTTPS支持CoAP/UDP低带宽

规范演进路径

1

SGP.01/02 (M2M Legacy)

2013年，局限性：SMS强依赖、SM-SR耦合、复杂度高

2

SGP.21/22 (Consumer)

2016年，创新：LPA、SM-DS发现服务、用户Pull模式

3

SGP.31/32 (IoT Modern)

2022-2024年，融合消费级简洁 + eIM服务器管理 + Push模式

📝 总结与选型

场景推荐规范原因
📱 手机/平板/手表SGP.21/22用户交互、扫码激活
⌚ 智能手表(无完整UI)SGP.21/22 或 SGP.31/32可用LPA或IPA
🚗 车联网/工业网关SGP.31/32服务器管理、大规模
🔋 智能电表/传感器SGP.31/32低功耗、无UI、NB-IoT
