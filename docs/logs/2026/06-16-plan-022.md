## 决策日志

### 1. 选 Cloudflare quick tunnel 不选 named tunnel
**决定**: quick tunnel (`--url` 模式)
**理由**:
- 用户明确"不用注册啥的"（quick tunnel 0 账号）
- "url 变化无所谓"（接受 ephemeral URL）
- 1 命令搞定（`cloudflared tunnel --url http://localhost:8766`）

### 2. cloudflared 装到 `~/.local/bin/` 不装到 `/usr/bin/`
**决定**: 用户级别 PATH 目录
**理由**:
- 没 sudo 权限（apt 要 sudo，命令行交互密码）
- `~/.local/bin/` 在 PATH 优先，行为等价
- 解包 .deb 提取 binary 比 curl GitHub release 快（GitHub 连接超时）

### 3. 不集成 tunnel 到现有 `bin/serve.sh`
**决定**: 写独立的 `bin/tunnel.sh` + `bin/tunnel-url.sh`
**理由**:
- serve.sh 管理 Flask（项目核心），tunnel 是辅助工具，分开管
- 用户能选 `bin/serve.sh` 不开 tunnel（本地模式）或 `bin/tunnel.sh` 开 tunnel
- 接口镜像（`start/--stop/--status/--url`）让用户无缝切换

### 4. 保留 Plan 021 Render 产物
**决定**: 不删 `render.yaml` + `gunicorn`
**理由**:
- 未来想切回 Render 不需要重建
- 0 维护成本（保留但不触发）
- 写 log 说明 Render 是 fallback option
