## 状态

`completed`（09:51 端到端通过：HTML/CSS/JS 改完，4 个新测试通过，Flask 重启新模板生效）

## 触发

09:37 飞书群用户（ou_50da...）发：*「在赛程表的最后显示一下这个top8列表，点刷新能刷新数据」*。上下文是上一轮 Plan 029/031 + Plan 041 已经把 `/api/qualification` 和 `best_3rd_race.rankings` 算好，但前端没有一站式 Top 8 视图。

## 设计决策

- **位置**：放在 `</main>` 与 `<footer>` 之间的独立 `<section id="third-place-top8-panel">`，**不依赖 tab**——Matches 和 Bracket 都能看到，刷新按钮已绑 `loadMatches()` 触发整个页面的数据重拉。
- **数据源**：复用既有 `/api/qualification` → `best_3rd_race.rankings`（12 支，按 FIFA 顺序排好），不改 API。
- **视觉**：表格化，1-8 名浅绿底 + ✅ 晋级，9-12 名浅灰底 + ❌ 淘汰，中间一条 dashed 分隔线。紧凑列宽适配桌面看板。
- **状态行**：每行额外列 `状态` 显式打标，避免用户误读。
- **空/失败态**：ranking 空 / qualification null → 显示「暂无第 3 名数据 · 数据更新后再次刷新」；timestamp 走 qualification cache 的 `generated_at`，北京时区格式化。
