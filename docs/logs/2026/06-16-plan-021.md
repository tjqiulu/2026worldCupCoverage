## 决策日志

### 1. 选 Render（不选 Railway/Fly.io）
**决定**: Render
**理由**: Blueprint 模式 0 配置，GitHub push 触发自动 redeploy 最丝滑；用户已熟悉 dashboard 类操作

### 2. 用 gunicorn `--workers 2 --timeout 120`
**决定**: workers=2 timeout=120
**理由**:
- Render free tier 0.1 CPU，workers=2 适配
- worldcup26.ir API 调 30s + 余量，timeout 120 留 4x buffer
- 单用户低流量配置已够

### 3. `PORT` 读 env var，默认 8766
**决定**: `int(os.environ.get("PORT", 8766))`
**理由**:
- Render 注入 `PORT=10000`，gunicorn 自动 bind
- 本地不设 env 时 fallback 8766（**与改前完全等价**）
- 满足"本地兼容不变"硬约束

### 4. 接受 Render ephemeral disk
**决定**: 接受 details.json 重启丢
**理由**:
- MVP 哲学（YAGNI）
- 比赛日刷新按钮会重写 details
- 真要持久化再升级 SQLite + persistent disk（plan 范围外）

### 5. Audit 脚本 G6 timeout 调整
**问题**: G6 第一次跑失败，`/api/matches` 报 timeout
**根因**: 5s timeout 太短，worldcup26.ir enrichment 实际 17s
**决定**: timeout 5s → 30s（G6）
**结果**: 重跑 8/8 PASS
