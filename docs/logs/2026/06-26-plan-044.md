## 设计决策

### 1. 锁定算法修正（src/data/qualification.py）

两处改：

- `min_pts > eighth_max + 3` → `min_pts > eighth_max`：原版要求 3 分缓冲，太保守
- `count_better < 7` → `count_better < 8`：原版要求 rank ≤ 7，应是 ≤ 8

修正后当前数据锁定 B/E/F（4 分全打完）。D（4 分）未锁定——理论上 L/J/H/G/K 全部 max 出能把 D 挤到第 9（概率 1/243 ≈ 0.4%）。

### 2. FIFA 配对矩阵（新文件 src/data/fifa_bracket_matrix.py）

硬编码 FIFA 2026 官方 R32 第 3 名配对：

```python
R32_3RD_OPPONENT_SETS = {
    "A": {"C","E","F","H","I"},  # 1A (Mexico) 从这 5 组选
    "B": {"E","F","G","I","J"},  # 1B (Canada)
    "D": {"B","E","F","I","J"},  # 1D (USA)  ← 关键
    "E": {"A","B","C","D","F"},  # 1E (Germany)
    "G": {"A","E","H","I","J"},
    "I": {"C","D","F","G","H"},
    "K": {"D","E","I","J","L"},
    "L": {"E","H","I","J","K"},
}
```

`BRACKET_ORDER = ["A","B","D","E","G","I","K","L"]` 贪心顺序：每轮找备选集中 FIFA 优先级最高且未分配的 3rd。

### 3. API 集成（src/app.py）

`_build_r32_resolution()` 读 qualification cache，调 FIFA 矩阵；`_resolve_r32_match()` 识别 R32 比赛形态（1X vs 3Y/Z 或 "USA" vs 3Y/Z 这种 host-nation 形态）挂上 `r32_resolved_opponent` 字段。

host-nation 形态用已有的 `build_team_id_map()`（Plan 027/028）做 "USA" → team_id → group letter 链路。

### 4. 前端渲染（src/static/js/main.js）

`resolveTeamForMirror()` 加 R32 resolved 分支：3X/Y/Z 形态优先用 `m.r32_resolved_opponent`，state 决定 CSS class（locked → 绿底，pending → 琥珀底）。
