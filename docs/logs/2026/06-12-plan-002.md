## 决策日志

### 1. 端口改 8766
**问题**: 8765 被本机其他服务（`ui/app.py --port 8765`）占着，curl 拿到的是 FastAPI 风格的响应
**决定**: 改用 8766
**记录**: 在 `src/app.py` 注释里写明原因

### 2. 不用 venv
**问题**: 系统 `python3-venv` 包未装，`apt install` 需 sudo 密码，`ensurepip` 也不可用
**决定**: 用 `pip install --user` 用户级安装
**影响**:
- 所有调用走 `python3 -m pip`（PATH 里 user pip 不一定可执行）
- pytest 用 `pyproject.toml` 的 `pythonpath = ["."]` 解决 src/ 导入
- Flask app 在 `app.py` 顶部手动加 `sys.path` 以支持 `python3 src/app.py` 直跑

### 3. ICS SUMMARY 用全名（不是 3 字母码）
**问题**: 我预期 "MEX vs RSA"，实际是 "Mexico vs South Africa"
**决定**: parser 改用全名解析 + 从 DESCRIPTION 提取 Group/Matchday
**影响**:
- 删掉原 `_parse_summary` 的 3 字母正则，改全名正则
- 加 `_parse_matchday` 和 `_parse_group` 处理 DESCRIPTION
- 加 `_parse_knockout_stage` 处理淘汰赛（"Round of 16", "Quarter-finals" 等）
- data 结构从 `{"code": "MEX"}` 改成 `{"name": "Mexico"}`
- country code 留给 Plan 003（要做 48 队映射表）

### 4. 淘汰赛队伍是占位符
**观察**: 真实 ICS 淘汰赛用 "W86 vs W88"（86 场胜者 vs 88 场胜者）/"2A vs 2B"（A 组第二 vs B 组第二）
**决定**: Plan 002 保留原文，Plan 003 不强行翻译（要等小组赛踢完才知道）
**降级**: UI 标签显示 "TBD vs TBD (R16)"，清晰表达"还没确定"

### 5. 缓存策略
**决定**: ICS 缓存 1 小时，过期重新拉
**理由**: 用户说"刷新即可"，1 小时 TTL 是合理默认

## 这次执行的最大收获

1. **真实数据 vs fixture 数据差异大**——预期 "MEX vs RSA"，实际 "Mexico vs South Africa"。**永远先跑真实数据再写测试**，否则测的是错误假设
2. **环境依赖要早探**——venv 不可用让我多花了 3 分钟。**AGE 建议每个 plan 开始前先做 1-2 分钟的 "environment probe"**
3. **端口冲突要主动检查**——`ss -tln` 永远先看，特别是 8000/5000/8765 这种常见端口
4. **pyproject.toml 的 pythonpath 救了测试**——避免每个测试都加 sys.path hack
5. **真实 ICS 的 DESCRIPTION 是个金矿**——`Matchday N\nGroup: Group X\nVenue: Y` 比 SUMMARY 信息多得多
6. **sys.path 显式管理在 app.py**——比依赖 PYTHONPATH env var 更鲁棒
