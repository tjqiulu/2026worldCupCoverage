## 关键发现（按 AGE 走 5-Why）

**Why 1**：截图 R32 卡 1B / 2B 位置是 FIFA 占位符，不是真队名。
**Why 2**：前端 `resolveBracketPlaceholder()`（main.js:145）查 `locked_top2` / `favored_top2` 都没匹配到。
**Why 3**：`/api/qualification` 返回的 B 组 4 队**全在 `pending`**，locked_top2=[]，eliminated=[]。
**Why 4**：`src/data/qualification.py:compute_per_group` 主循环在 `all_played=True` 时，所有 `if not all_played` 守卫把判定全跳过了，所有队走 else 分支进 pending。
**Why 5 (root)**：Plan 029 设计时把 `locked_top2` 定义成"未踢完但数学严格锁定前 2"，没有"已踢完 = 100% 确定前 2"分支。语义错位——用户视觉期望 `locked_top2 = 32 强卡里渲染真队的所有情况`。

**测试反映错误语义**：`tests/test_qualification.py::TestLockedTop2::test_all_finals_no_locked` 硬编码 `assert len(result["locked_top2"]) == 0  # 全踢完了，没"待锁定"`，把错误语义锁进了测试。`TestEliminated::test_all_played_no_eliminated` 同样问题。

## 修复

### 修 `src/data/qualification.py:compute_per_group`（+18 行）

主循环结束后加 `if all_played and len(standings) >= 4` 分支：

- `pending.clear()`（避免主循环污染）
- standings[:2] → 追加到 `locked_top2`（防御性 `any(...)` 跳过重复）
- standings[2:] → 追加到 `eliminated`
- `favored_top2` 不动（favored 本来就是"未踢完但软"概念，已踢完没软状态）

standings 已按 FIFA 优先级（pts desc / gd desc / gf desc）排好序（`compute_standings_from_details`），所以 index = 最终排名。

### 翻 2 个 + 加 1 个测试

- `test_all_finals_no_locked` → `test_all_finals_played_top2_locked_bot2_eliminated`：断言 `locked_ids == {top 2}` + `eliminated_ids == {bot 2}` + `pending == []`
- `test_all_played_no_eliminated` → `test_all_played_bot2_in_eliminated`：断言 `bot 2 in eliminated`
